Welcome to BALLS application.

This game may appear to be kind of difficult to implement, but is just 10 lines of code, really.
It is fun and you can learn how to do cool stuff very easy. As used before, we have our setup() method in which we declare the size of the window and the background. Then, we have our draw() method in which we call the variableEcllipse() function and we send the current mouse position and the previous mouse position- each ball is drawed accordingly to the speed of the mouse between 2 different positions. That’s why we need this information.
In the variableEcllipse() function we calculate the speed between the current mouse position and the previous mouse position. Then, the speed influences the color of the border of the ball as well as the size of the ellipse.
A slower move means a small ellipse, while moving the mouse quickly determines the ellipse to be drawn large. So we use the stroke(speed) attribute to set the border of the ellipse, and the ellipse(x,y,speed,speed) to draw a ball at the specific x and y coordinates with the size “speed”.
